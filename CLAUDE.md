Contact Form — Google Sheets + Email Notification
Architecture
[Website Form] → POST JSON → [Google Apps Script Web App] → [Writes to Google Sheet + Sends email]
No backend server required. Google Apps Script is the free middleware.
Google Apps Script Code
The site owner creates a Google Sheet called "Website Contact Form" with headers:
| Timestamp | Name | Email | Message | Language |
Then adds this Apps Script (Extensions → Apps Script):
javascriptconst SHEET_NAME = "Sheet1";
const NOTIFY_EMAIL = "manuchehr@ghafforzoda.net";

function doPost(e) {
  try {
    const data = JSON.parse(e.postData.contents);
    const sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName(SHEET_NAME);

    // Honeypot check — reject bots silently
    if (data.website) {
      return ContentService
        .createTextOutput(JSON.stringify({status: "success"}))
        .setMimeType(ContentService.MimeType.JSON);
    }

    // Validate required fields
    if (!data.name || !data.email || !data.message) {
      return ContentService
        .createTextOutput(JSON.stringify({status: "error", message: "Missing required fields"}))
        .setMimeType(ContentService.MimeType.JSON);
    }

    // Write to sheet
    sheet.appendRow([
      new Date().toISOString(),
      data.name,
      data.email,
      data.message,
      data.language || "en"
    ]);

    // Send email notification
    const subject = "New website inquiry from " + data.name;
    const body = "Name: " + data.name + "\n"
               + "Email: " + data.email + "\n"
               + "Language: " + (data.language || "en") + "\n\n"
               + "Message:\n" + data.message;

    MailApp.sendEmail(NOTIFY_EMAIL, subject, body, {
      replyTo: data.email
    });

    return ContentService
      .createTextOutput(JSON.stringify({status: "success"}))
      .setMimeType(ContentService.MimeType.JSON);

  } catch (error) {
    return ContentService
      .createTextOutput(JSON.stringify({status: "error", message: error.toString()}))
      .setMimeType(ContentService.MimeType.JSON);
  }
}

<script>
const FORM_ENDPOINT = "PASTE_GOOGLE_APPS_SCRIPT_WEB_APP_URL_HERE";

document.getElementById("contact-form").addEventListener("submit", async function(e) {
  e.preventDefault();

  const form = e.target;
  const status = document.getElementById("form-status");
  const button = form.querySelector("button[type='submit']");

  button.disabled = true;
  button.textContent = "Sending... / Отправка...";
  status.textContent = "";

  const payload = {
    name: form.name.value.trim(),
    email: form.email.value.trim(),
    message: form.message.value.trim(),
    website: form.website.value, // honeypot
    language: document.documentElement.lang || "en"
  };

  try {
    const response = await fetch(FORM_ENDPOINT, {
      method: "POST",
      body: JSON.stringify(payload),
      headers: { "Content-Type": "text/plain" }
      // text/plain avoids CORS preflight with Apps Script
    });

    const result = await response.json();

    if (result.status === "success") {
      status.textContent = "Message sent! / Сообщение отправлено!";
      status.className = "success";
      form.reset();
    } else {
      status.textContent = "Error. Please try again. / Ошибка. Попробуйте снова.";
      status.className = "error";
    }
  } catch (err) {
    status.textContent = "Error. Please email manuchehr@ghafforzoda.net directly.";
    status.className = "error";
  } finally {
    button.disabled = false;
    button.textContent = "Send / Отправить";
  }
});
</script>
Critical Implementation Notes

Use Content-Type: text/plain NOT application/json — avoids CORS preflight
Honeypot field must be display:none with tabindex="-1"
Show loading state on button during submission
Clear form on success
On error, show fallback email address
Client-side validation: name not empty, email format valid, message not empty


Language Toggle

ghafforzoda.net — Design Specification
Aesthetic direction: Modern startup — inspired by Anthropic.com and Figma.com.
One-word brief: Editorial.
Feel: Clean, confident, spacious. Like it belongs to someone who thinks clearly and works with precision. Not corporate, not flashy.

Color Palette
Background
Warm off-white/cream — NOT pure white.

Primary background: #FAFAF8 or #F7F6F3
This warmth is what separates a premium site from a generic one

Text

Primary text: #1A1A1A (near-black, NOT pure black)
Secondary text: #6B6B6B

Accent — Warm Terracotta (DECIDED)

Primary accent: #C2613A
Hover/active: #A8522F

CTA buttons (filled)
CTA button outlines (secondary)
Link hover states

Everything else is neutral (text colors + cream background).
DO NOT use

Purple gradients
Bright blues
Neon greens
Gradient backgrounds
Rainbow palettes
Dark mode (not needed for v1)


Typography
CRITICAL: Do NOT use Inter, Roboto, Arial, or system fonts.
These are the #1 sign of generic AI-generated sites.
Recommended Pairing
Headlines — geometric or humanist sans-serif with character:

Satoshi (free via Fontshare — modern, slightly geometric, similar to Figma)
General Sans (free via Fontshare — clean but distinctive)
Cabinet Grotesk (free via Fontshare — bold, startup feel)

Body text — clean, highly readable:

DM Sans (free Google Font — pairs well with geometric headlines)
Source Serif 4 (free Google Font — elegant serif, good for About section)
Pairing serif body with sans-serif headlines creates editorial feel

Performance

Fonts: preload main weights only
Target: < 2 second load time on 3G
No heavy JS libraries for animations (use CSS transitions + IntersectionObserver)
